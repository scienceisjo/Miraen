처음이라면 — Supabase 테이블 2개 만들기 SQL 보기 
-- 1) 상황별 대피 시나리오 카드
create table if not exists eq_manual_cards (
  id uuid primary key default gen_random_uuid(),
  room text not null, author text,
  type text, evac text, prompt text, revise text, reason text, img text,
  updated_at timestamptz default now());
alter table eq_manual_cards enable row level security;
create policy "classroom_open" on eq_manual_cards for all using (true) with check (true);
alter publication supabase_realtime add table eq_manual_cards;

-- 2) 미래엔 디지털 지도(지진·화산 데이터) 업로드
create table if not exists eq_quake_data (
  id uuid primary key default gen_random_uuid(),
  room text not null, author text, num int,
  period text, kind text, note text, img text,
  updated_at timestamptz default now());
alter table eq_quake_data enable row level security;
create policy "classroom_open_data" on eq_quake_data for all using (true) with check (true);
alter publication supabase_realtime add table eq_quake_data;

우리 반이 함께 만드는 우리 동네 지진 안전 매뉴얼

이 매뉴얼은 선생님이 주는 게 아니라, 우리가 함께 만듭니다. 한 사람이 전부 할 수는 없어요. 누구는 정해진 기간의 지진·화산 데이터를 조사해 지도에 올리고, 어떤 모둠은 '집에서', 어떤 모둠은 '학교에서', 또 어떤 모둠은 '새벽에' 같은 상황을 하나씩 맡습니다. 이 조각들이 모두 모여야 비로소 우리 반의 매뉴얼 한 권이 완성돼요.

🤝
의존내 조각이 빠지면 전체 지도와 매뉴얼에 구멍이 생겨요. 친구의 조각도, 내 조각도 서로에게 필요해요.
✅
책임내가 맡은 기간·상황은 내가 끝까지 책임지고 채워요. 내 역할을 잘 해내야 우리 반 매뉴얼이 가능해집니다.
🙋
참여한 명도 빠짐없이 한 조각씩. 모두의 손이 닿아야 진짜 '우리 반'의 매뉴얼이에요.
함께 배우고, 함께 문제를 해결하고, 함께 성장해요. 네 역할이 곧 우리 반의 안전이야!

입장하기

실시간 · 반 · 나

1지진·화산 데이터 조사 & 미래엔 지도 업로드

내 번호 -번 담당 기간 → 번호를 입력해 입장하세요
전체 번호별 기간표 보기
활동 순서 ① 기상청에서 내 기간의 지진·화산을 조사 → ② 미래엔 디지털 지도(세계지도)에서 발생 지역을 선택해 좌표 등록 → ③ 지도를 다운로드 → ④ 아래에 업로드해 우리 반 지도로 합치기

2상황별 대피 시나리오 만들기

우리 모둠이 맡은 상황(집·학교·학원·영화관·새벽 등)을 하나 골라 카드를 채워요. [＋ 상황 카드]로 추가합니다.

아직 카드가 없어요.

📖 우리 반 종합 매뉴얼

조사한 지도와 모둠별 시나리오가 모여 하나의 매뉴얼이 됩니다.

내용이 올라오면 여기에 모입니다.
